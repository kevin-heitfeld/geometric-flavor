# Contributing to Geometric Flavor

Thank you for your interest in contributing! This project welcomes contributions from:

- **Theoretical physicists**: Validate results, extend to other models
- **String theorists**: Build explicit Calabi-Yau constructions
- **Phenomenologists**: Test predictions, improve fits
- **Computational scientists**: Optimize code, add features

---

## How to Contribute

### Reporting Issues

Found a bug or have a question?

1. Check existing [Issues](../../issues) first
2. Open a new issue with:
   - Clear description
   - Steps to reproduce (if bug)
   - Expected vs actual behavior
   - Your environment (Python version, OS)

### Suggesting Enhancements

Have an idea for improvement?

1. Open an issue with tag `enhancement`
2. Describe the feature and motivation
3. Discuss implementation approach

### Code Contributions

Ready to contribute code?

1. **Fork** the repository
2. **Create** a feature branch: `git checkout -b feature/amazing-feature`
3. **Make** your changes with clear commit messages
4. **Test** thoroughly: run all scripts, check outputs
5. **Document**: Update README and add docstrings
6. **Submit** a pull request

---

## Code Standards

### Python Style

- Follow PEP 8 style guide
- Use descriptive variable names
- Add docstrings to functions
- Comment complex physics calculations

### Physics Standards

- Cite papers for formulas (arXiv numbers)
- Use standard PDG values for experimental data
- Document approximations and assumptions
- Validate against known results

### Testing

Before submitting:
- Run key scripts: `stress_test_k_patterns.py`, `explain_n_ordering.py`
- Check figures are generated correctly
- Verify numerical accuracy
